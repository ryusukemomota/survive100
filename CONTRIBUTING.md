# Contributing to SURVIVE 100

SURVIVE 100への貢献をありがとうございます！このドキュメントでは、プロジェクトへの貢献方法について説明します。

## 🚀 はじめに

### 開発環境のセットアップ

1. リポジトリをフォーク
2. ローカルにクローン
```bash
git clone https://github.com/your-username/survive-100.git
cd survive-100
```

3. 依存関係をインストール
```bash
npm install
```

4. 開発サーバーを起動
```bash
npm run dev
```

## 📝 コントリビューションの種類

### バグ報告
- GitHubのIssuesでバグを報告
- 再現手順を詳細に記載
- 期待される動作と実際の動作を明記

### 機能提案
- 新機能のアイデアをIssuesで提案
- ユースケースと実装案を含める
- 健康教育の観点からの価値を説明

### コード貢献
- バグ修正
- 新機能の実装
- パフォーマンス改善
- テストの追加

## 🔧 開発ガイドライン

### コーディング規約
- TypeScriptを使用
- ESLintとPrettierの設定に従う
- コンポーネントは関数型で記述
- 適切な型定義を行う

### コミットメッセージ
```
type(scope): description

例:
feat(game): add new disease system
fix(ui): resolve mobile layout issue
docs(readme): update installation guide
```

### ブランチ命名規則
- `feature/機能名`
- `fix/バグ名`
- `docs/ドキュメント名`

## 🧪 テスト

### テストの実行
```bash
npm run test
```

### テストの追加
- 新機能には必ずテストを追加
- バグ修正には回帰テストを追加
- ゲームロジックの重要な部分は必ずテスト

## 📋 プルリクエストのプロセス

1. **ブランチ作成**
```bash
git checkout -b feature/your-feature-name
```

2. **変更の実装**
- 小さく、焦点を絞った変更を心がける
- コミットメッセージは明確に

3. **テストの実行**
```bash
npm run test
npm run lint
```

4. **プルリクエストの作成**
- 変更内容を詳細に説明
- 関連するIssueがあれば参照
- スクリーンショットがあれば添付

5. **レビューの対応**
- フィードバックに迅速に対応
- 必要に応じて追加のコミットを行う

## 🎯 優先度の高い貢献領域

### 高優先度
- バグ修正
- パフォーマンス改善
- アクセシビリティの向上
- モバイル対応の改善

### 中優先度
- 新しい病気の追加
- UI/UXの改善
- 多言語対応
- 実績システムの拡張

### 低優先度
- 新機能の追加
- 実験的な機能
- 大規模なリファクタリング

## 🏥 健康教育の観点

SURVIVE 100は健康教育を目的としたゲームです。貢献する際は以下を考慮してください：

- **科学的根拠**: 医学的に正確な情報を基にする
- **教育効果**: プレイヤーの健康意識向上に寄与する
- **バランス**: ゲーム性と教育性のバランスを保つ
- **アクセシビリティ**: 幅広い年齢層が理解できる内容

- Issuesで質問
- メールでの直接連絡
